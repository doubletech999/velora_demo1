المطلوب: تعديل Laravel Backend و Admin Panel لدعم فكرة "المسارات الجديدة" حيث المسارات والتخييم هي المحتوى الأساسي وليس الأماكن السياحية.

الفكرة الأساسية:
- المسارات (Routes) - type='route' هي المحتوى الأساسي
- التخييم (Camping) - type='camping' جزء من المحتوى الأساسي  
- الأماكن السياحية (Tourist Sites) - type='site' محتوى ثانوي

التعديلات المطلوبة:

1. Database:
   - إضافة عمود type في جدول sites: enum('type', ['site', 'route', 'camping'])->default('site')
   - تحديث البيانات الموجودة: المسارات التي تحتوي على hiking أو length>=5 → type='route'
   - التخييمات التي تحتوي على camping → type='camping'
   - الباقي → type='site'

2. API Controller (/api/sites):
   - دعم filter: ?type=route أو ?type=camping أو ?type=site
   - دعم pagination
   - دعم search
   - إرجاع type في response
   - إرجاع guide_name و guide_name_ar في response الرئيسي

3. API Resource:
   - إضافة type في response
   - إضافة guide_name و guide_name_ar
   - إضافة guide object مع user relation
   - جميع الحقول المطلوبة: id, name, name_ar, description, type, location, latitude, longitude, images, length, estimated_duration, difficulty, activities, rating, review_count, price, guide_id, guide_name, guide_name_ar, guide

4. Admin Panel:
   - إضافة Filter Tabs: المسارات، التخييم، الأماكن السياحية، الكل
   - إضافة Type field في Create/Edit form (dropdown: route, camping, site)
   - إضافة Badge في Table لعرض النوع
   - إضافة Statistics في Dashboard (عدد المسارات، التخييمات، الأماكن السياحية)
   - جعل المسارات والتخييم بارزة (الأساسية)

5. Validation:
   - type: required|in:site,route,camping
   - جميع الحقول الأخرى حسب الحاجة

النتيجة المتوقعة:
- المسارات (type='route') هي المحتوى الأساسي في التطبيق
- التخييم (type='camping') جزء من المحتوى الأساسي
- الأماكن السياحية (type='site') محتوى ثانوي
- Admin Panel يدعم التصنيف الجديد
- API يدعم التصنيف الجديد
- Flutter App يعمل بشكل صحيح

Endpoints المطلوبة:
- GET /api/sites?type=route → المسارات
- GET /api/sites?type=camping → التخييم
- GET /api/sites?type=site → الأماكن السياحية
- GET /api/sites → جميع المواقع

Response Format:
{
  "data": [
    {
      "id": 1,
      "name": "اسم الموقع",
      "name_ar": "اسم الموقع بالعربية",
      "type": "route", // أو "camping" أو "site"
      "guide_name": "اسم المرشد",
      "guide_name_ar": "اسم المرشد بالعربية",
      "guide": { ... },
      ...
    }
  ],
  "current_page": 1,
  "last_page": 5,
  "per_page": 10,
  "total": 50
}
